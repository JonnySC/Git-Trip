Introducción a Git

En la sección de introducción, exploraremos qué es Git, cómo funciona y las herramientas esenciales para su implementación en el desarrollo de proyectos.

  • Qué es Git y cómo funciona:
    -Git es un sistema de control de versiones distribuido.
    -Gestiona eficientemente el historial de cambios en un proyecto.
    -Su naturaleza distribuida permite que cada colaborador tenga una copia completa del repositorio.
  • Tooling para desarrollo con Git:

    Antes de sumergirse en el uso práctico de Git, es esencial comprender las siguientes herramientas:

    • Terminal: Si prefieres trabajar en la línea de comandos, puedes utilizar la terminal para ejecutar comandos Git directamente. Algunos comandos básicos incluyen git init, git clone, git add, git commit y git push.
    • Servicios de alojamiento de Git: Plataformas como GitHub, GitLab, Bitbucket y SourceForge ofrecen servicios de alojamiento para tus repositorios Git. Puedes crear, colaborar y compartir proyectos con otros desarrolladores utilizando estas plataformas.
    • Editores de texto: Aunque no es específico de Git, tener un buen editor de texto es crucial para escribir y editar código. Algunos editores populares son Visual Studio Code, Sublime Text, Atom y Vim.

    Recuerda que elegir las herramientas adecuadas depende de tus preferencias personales y del flujo de trabajo de tu equipo. ¡Explora y encuentra las que mejor se adapten a ti!

Conceptos Fundamentales de Git

Antes de sumergirse en el uso práctico de Git, es esencial comprender los siguientes conceptos:

  • Repositorios: Un repositorio es un almacén de código fuente que contiene todos los archivos y la historia de cambios de un proyecto. Puede ser local (en tu máquina) o remoto (en un servidor). Los repositorios remotos son comunes en plataformas como GitHub, GitLab o Bitbucket.
  • Commits: Un commit representa un conjunto de cambios realizados en el código. Cada commit tiene un mensaje descriptivo que explica qué se modificó. Los commits forman el historial de cambios del proyecto.
  • Ramas: Las ramas son líneas de desarrollo independientes dentro del repositorio. Permiten trabajar en nuevas funcionalidades o correcciones sin afectar la rama principal (generalmente llamada "master" o "main"). Por ejemplo, puedes crear una rama para desarrollar una nueva característica y luego fusionarla con la rama principal.
  • Etiquetas: Las etiquetas son puntos específicos en la historia del repositorio. Se utilizan para marcar versiones estables o hitos importantes. A menudo se usan para identificar versiones de lanzamiento.
  • Remotos: Los remotos son repositorios alojados en servidores externos (como GitHub o GitLab). Facilitan la colaboración y el intercambio de código entre desarrolladores. Puedes clonar, empujar y tirar cambios hacia y desde repositorios remotos.

Recuerda que estos conceptos son esenciales para comprender cómo funciona Git y cómo aprovecharlo al máximo en tu flujo de trabajo. ¡Buena suerte con tu presentación!

Comandos básicos de Git

  • git init: Crea un nuevo repositorio local de Git. Puedes ejecutarlo en el directorio actual para iniciar un repositorio o especificar un nombre de proyecto para crear un repositorio en un directorio nuevo.
  • git clone: Copia un repositorio existente. Si el repositorio está en un servidor remoto, utiliza la siguiente sintaxis: git clone nombredeusuario@host:/path/to/repository. Para clonar un repositorio local, simplemente usa: git clone /path/to/repository.
  • git add: Agrega archivos al área de preparación (staging). Por ejemplo, para indexar el archivo temp.txt, ejecuta: git add temp.txt.
  • git commit: Crea una instantánea de los cambios y los guarda en el directorio Git. Acompaña el commit con un mensaje descriptivo: git commit -m "Mensaje del commit aquí".
  • git config: Establece configuraciones específicas de usuario, como el correo electrónico y el nombre de usuario. Por ejemplo:
    • Configurar un email global: git config --global user.email tuemail@ejemplo.com.
    • Configurar un email local para un repositorio específico: git config --local user.email tuemail@ejemplo.com.
  • git status: Muestra la lista de archivos modificados, así como los archivos preparados o confirmados.

Flujo de trabajo con Git

  1. Creación de una Rama de Funcionalidad:

    Cada nueva funcionalidad o corrección de errores se desarrolla en una rama separada.

    Ejemplo: feature/nueva-funcionalidad.

  2. Desarrollo y Confirmación:

    Realiza cambios en la rama de funcionalidad.

    Confirma los cambios con mensajes descriptivos.

  3. Solicitud de Extracción (Pull Request):

    Abre una solicitud de extracción para fusionar la rama de funcionalidad con main.

    Revisa los cambios y colabora con otros miembros del equipo.

  4. Revisión y Aprobación:

    Otros revisan el código y proporcionan comentarios.

    Una vez aprobada, se fusiona la rama de funcionalidad con main.

  5. Despliegue:

    Los cambios confirmados en main se despliegan en producción.

Buenas prácticas con Git

  1. Escribe Mensajes de Commit Significativos:

    Usa el verbo imperativo en tus mensajes de commit (por ejemplo, "Add", "Change", "Fix", "Remove") si necesitas mas información puedes vistar este sitio https://www.conventionalcommits.org/en/v1.0.0/.

    Limita el mensaje a 50 caracteres para el título del commit.

    Proporciona contexto adicional en el cuerpo del mensaje si es necesario.

  2. Crea Ramas para Cada Funcionalidad o Corrección:

    Desarrolla cada nueva funcionalidad o corrección en una rama separada.

    Por ejemplo: feature/nueva-funcionalidad.

  3. Actualiza Constantemente tu Repositorio Local:

    Utiliza git pull para mantener tu repositorio local al día con los cambios remotos.

  4. Solicita Extracciones (Pull Requests):

    Abre una solicitud de extracción para fusionar tus cambios en la rama principal (main).

    Colabora con otros miembros del equipo y revisa los cambios.

  5. Evita Commits que Hacen Demasiadas Cosas:

    Si un commit hace demasiadas cosas, considera dividirlo en varios commits más pequeños.

  6. Comparte el Contexto Necesario en el Cuerpo del Mensaje de Commit:

    Si un commit necesita más contexto, añádelo en el cuerpo del mensaje.

Herramientas y recursos adicionales con Git

Extensiones de Git para Visual Studio Code

  • GitLens: Con más de 25 millones de instalaciones, GitLens es la extensión Git más popular para Visual Studio Code. Proporciona información valiosa sobre la autoría del código y desbloquea todo el poder de Git en VS Code. Te permite ver la historia de Git, la diferencia de archivos confirmados, la información de culpabilidad (blame) y más.
  • Git History Diff: Esta extensión te permite ver la historia de Git, los detalles de los commits, la diferencia de archivos confirmados y la información de culpabilidad (blame). Puedes comparar ramas, ver la historia de archivos o carpetas, y explorar cambios de línea. También puedes ver las diferencias entre ramas y comparar archivos no confirmados con un commit específico.
  • Git Graph: Git Graph te permite visualizar un gráfico de Git de tu repositorio y realizar acciones de Git directamente desde el gráfico. Puedes ver la historia de ramas, archivos, líneas y autores. También muestra detalles de commits, diferencias y stashes.

En resumen, estas extensiones te ayudarán a trabajar con Git de manera más eficiente dentro de Visual Studio Code. ¡Explóralas y elige las que mejor se adapten a tus necesidades! 😊
